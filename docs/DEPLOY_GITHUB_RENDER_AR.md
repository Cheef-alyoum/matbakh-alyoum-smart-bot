# GitHub + Render — المسار الموصى به

## المسار
1. مشروعك على GitHub
2. GitHub Actions يفحص النسخة عند كل push
3. Render يسحب من الفرع الرئيسي وينشر تلقائيًا

## ما الذي يفعله GitHub Actions هنا؟
- يفحص وجود الملفات الأساسية
- يشغّل اختبارًا دخانيًا محليًا
- يمنعك من دفع نسخة مكسورة بسهولة

## الترتيب الموصى به للفروع
- `main` = إنتاج
- `staging` = تجارب قبل الإنتاج

## السياسة العملية
- أي تعديل مهم على الرسائل أو المنيو أو الـ SEO جرّبه على `staging`
- بعد نجاح الفحص، انقله إلى `main`
- Render سينفذ Auto Deploy على `main`

## أسرار GitHub
هذه النسخة لا تحتاج أسرار GitHub لتعمل كفحص أساسي، لأن النشر الفعلي يتم من Render.
إذا أردت لاحقًا تنفيذ فحوص إضافية أو نشرات مخصصة، نضيف أسرارًا منفصلة.
