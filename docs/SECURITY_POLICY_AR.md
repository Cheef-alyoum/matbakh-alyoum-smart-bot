# الأمان والامتثال — النسخة النهائية

## القواعد المعتمدة
- عدم تضمين الأسرار أو مفاتيح الواجهة داخل GitHub.
- الاعتماد على متغيرات البيئة فقط للمفاتيح الحساسة.
- حفظ المحادثات والطلبات لأغراض التشغيل والتحليل الداخلي فقط.
- عدم مشاركة بيانات العملاء خارج الأرقام الإدارية المعتمدة.
- عدم إرسال رسائل تسويقية أو استبيانات إلا بموافقة العميل وبموافقة الإدارة.
- الحفاظ على أسلوب راقٍ غير مزعج في مخاطبة العميل، مع بيع وإقناع دون إلحاح.

## نقاط تشغيلية
- اجعل Webhook الخاص بواتساب محميًا بـ Verify Token صحيح.
- استخدم Service Role Key داخل Render فقط، ولا تضعه داخل الواجهة العامة.
- اجعل الدومين العام يقدّم صفحات HTML واضحة ويفصل واجهات الإدارة وAPI عن الفهرسة العامة.
- نفّذ نسخًا احتياطيًا دوريًا من قاعدة البيانات قبل أي reset شامل.
